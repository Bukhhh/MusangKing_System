Step-by-Step Processing Pipeline

A detailed academic visualization of the 8-stage hybrid segmentation and feature extraction logic used to classify the Durian.

{% if data and data.pipeline %}
1

Gamma Correction

Why: Normalization Applies V_out = V_in ^ 0.6. Enhances contrast to handle varying lighting conditions across the dataset.
2

LAB Color Space

Why: Color Separation Converted BGR to CIELAB. Separate Lightness (L) from Color (A,B) channels for robust clustering under shadows.
3

K-Means Clustering

Why: Segmentation Unsupervised clustering with K=3 (Background, Husk, Flesh). Groups pixels by color similarity.
4

Morphology

Why: Noise Removal Closing operation (5x5 kernel). Removes small holes and smooths boundaries for a clean mask.
5

Contour Detection

Why: Localization Finds all object boundaries. Selects the largest contour (green) as the primary object.
6

Feature Extraction

Why: Geometric Analysis Calculates Area, Perimeter, and Compactness. Visualize Bounding Box (Blue) and Convex Hull (Green).
7

Color Analysis

Why: Color Characterization Computes RGB Histograms to quantify the exact color distribution of the husk for species differentiation.
8

Binary Masking

Why: ROI Extraction The final clean binary mask. Used to isolate the fruit from the background for accurate prediction.
